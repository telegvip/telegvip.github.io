# Archivos que debes eliminar del repositorio anterior

No necesitas crear ni ejecutar un archivo BAT.

Antes de subir esta versión, elimina del repositorio de GitHub estos recursos visuales antiguos si todavía aparecen:

- `logo-telegvip-original.webp`
- `logo-telegvip-original.png`
- `logo-telegvip-icon.webp`
- `logo-telegvip-icon.png`
- `icono-telegvip.webp`

También elimina cualquier archivo `.zip` que hayas subido dentro del repositorio. GitHub Pages no necesita el ZIP; necesita los archivos descomprimidos.

## No elimines

- `index.html`
- `logo-telegvip.webp`
- `favicon.ico`
- `favicon-16x16.png`
- `favicon-32x32.png`
- `favicon-192x192.png`
- `favicon-512x512.png`
- `apple-touch-icon.png`
- `site.webmanifest`
- `firebase.json`
- `.firebaserc`
- `.nojekyll`
- Las reglas de Firebase ni los archivos Markdown de instrucciones.

## Forma recomendada de actualizar GitHub

1. Elimina los cinco logos antiguos indicados arriba.
2. Sube todos los archivos de esta nueva carpeta a la raíz del repositorio.
3. Confirma el reemplazo de `index.html`, favicons y `site.webmanifest`.
4. Espera uno o dos minutos y recarga la web con `Ctrl + F5` para evitar que el navegador muestre la versión guardada en caché.
